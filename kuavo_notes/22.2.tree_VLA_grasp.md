## 第 0 章：一句话结论

| 项目 | 22.1（原版） | 22.2（行为树版） |
|------|-------------|-----------------|
| NUC 终端 5 入口 | `vla_auto_grasp_daemon.py` | `vla_bt_daemon.py` |
| 编排方式 | Python 类 + `is_working` 状态机 + 一个超长函数 | py_trees：`Sequence` / `Selector` + 黑板 |
| 其余 8 个终端 | **完全相同** | **完全相同** |
| 底层 WBC/IK/夹爪 | 闭源 `.so` 执行，两者都不改底层 | 同左 |

**核心变化只在 NUC 终端 5**：把「状态机小脑」换成「行为树小脑」，抓取物理逻辑从 daemon 抽到 `bt/grasp_skills.py`，流程用树编排。

## 第 1 章：22.1 vs 22.2 详细对比

### 1.1 架构层面对比

```text
【22.1 状态机版】

  /vla/master_command
         │
         v
  command_callback()
         │  if is_working: return
         │  if action==grab:
         v
  execute_grasp_pipeline()  ← 一个函数从头到尾跑完（300+ 行）
         │  采点 → IK → 开爪 → 运动 → 关爪 → 抬升 → 收手
         v
  is_working = False


【22.2 行为树版】

  /vla/master_command
         │
         v
  _on_master_command()  →  写入黑板 vla/command
         │
         v
  主循环 tick_once() @ 10Hz
         │
         v
  Selector [VLA_ROOT]
    ├── HandleGrab  (Sequence)
    ├── HandleChat  (Sequence)
    └── Idle        (RUNNING)
```

### 1.2 功能对比表

| 维度 | 22.1 `vla_auto_grasp_daemon.py` | 22.2 `vla_bt_daemon.py` + `bt/` |
|------|--------------------------------|----------------------------------|
| **编排模型** | 隐式状态机：`is_working` 布尔锁 | 显式 BT：`SUCCESS/FAILURE/RUNNING` |
| **抓取流程** | `execute_grasp_pipeline()` 单函数 | `GraspPipeline` 子树，11 步独立节点 |
| **chat/TTS 分支** | 无（TTS 仅由 Orin 主控 HTTP 直调） | 树内 `HandleChat` → `SpeakTTS` |
| **忙碌锁** | `self.is_working` 手动置 True/False | 黑板 `vla/working` + `BusyScope` 自动解锁 |
| **失败解锁** | 各 `return` 前需手动 `is_working=False`，漏写会死锁 | `BusyScope` 子树结束必解锁 |
| **采 YOLO 点** | `while` 循环一次阻塞收满 10 帧 | 订阅缓存 + 增量采点（每 tick 取最新帧，非阻塞） |
| **YOLO 失败重试** | 失败需手动重发指令 | 失败自动清 command；需 Orin 重发或手动 pub |
| **YOLO 前置检查** | 无，直接等话题 | `CheckYoloAlive` 确认话题有发布者 |
| **grab 指令防抖** | 无 | 8s 内重复 grab 忽略（防 ASR 误触发） |
| **左/右 TCP 偏置** | 写死在 daemon 常量 | `bt/config.py` + ROS 参数可调 |
| **夹爪安全** | 部分脚本 `[90, 1.5]` | 统一 `claw_safe.py`（见 20.gripper_issue.md） |
| **失败定位** | 看 print 日志猜阶段 | 日志带节点名，如 `CloseClaw` FAILED |
| **启动可视化** | 无 | 启动打印 ASCII 行为树 |
| **可调参数** | 改源码常量 | ROS 私有参数（话题、超时、tick 频率） |
| **代码结构** | 单文件 ~285 行 | 多文件：`grasp_skills` / `nodes` / `trees` |
| **依赖** | rospy + kuavo_msgs | 同上 + `pip install py_trees` |
| **与 Orin 主控接口** | 订阅 `/vla/master_command` | **相同** |
| **与 YOLO 接口** | 订阅 `/vla/yolo_target` | **相同** |
| **与 IK/WBC 接口** | `/ik/two_arm_hand_pose_cmd_srv`、`/kuavo_arm_target_poses` | **相同** |
| **避障策略** | 关节空间 75° 大展翅（源码内） | 同逻辑，在 `grasp_skills.plan_grasp()` |

### 1.3 什么没变（两套方案共用）

以下内容 **22.1 和 22.2 完全一致**，换 BT 不影响：

1. **9 终端部署架构**（仅终端 5 脚本名不同）
2. **Orin 主控** `rtasr_python3_demo.py`：ASR → LLM → 发 `/vla/master_command` + HTTP TTS
3. **YOLO+TF2** 发布 `/vla/yolo_target`
4. **阵列一** WBC / IK / `look_down.py`
5. **TTS** `tts_server.py`（lab 用户、5000 端口）
6. **闭源底层**：`libhumanoid_wbc.so`、`libros_hardware_plant.so` 等不改、不碰
7. **22.1 文档里的硬件踩坑**：USB 麦克风、alsamixer、`disable_signals`、Ghost_words 等

### 1.4 什么时候用哪个

| 场景 | 推荐 |
|------|------|
| 日常联调、快速验证抓取 | 22.1 状态机版（更短、无 py_trees 依赖） |
| 要加 chat 分支、拆步骤、加重试/超时 | **22.2 行为树版** |
| 要学习/扩展 BT、对接更多 action | **22.2** |
| 要和官方 `kuavo_strategy_pytree` 对齐 | **22.2** |

**禁止**：同一时刻跑 `vla_auto_grasp_daemon.py` 和 `vla_bt_daemon.py`（会抢 `/vla/master_command` 和手臂控制权）。

## 第 2 章：py_trees 行为树结构（终端 5 内部）

### 2.1 完整树形图

```text
Selector [VLA_ROOT]
├── Sequence [HandleGrab]
│   ├── HasAction[grab]              ← 黑板 cmd.action == "grab"
│   ├── EnsureNotBusy                ← vla/working == False
│   ├── MarkBusy                     ← working = True
│   └── BusyScope [GrabBusyScope]    ← 成功/失败都自动 working = False
│                                      ← 失败时清 vla/command，防 YOLO 无限重试
│       └── Sequence [GrabWork]
│           ├── Sequence [GraspPipeline]
│           │   ├── ParseTarget      ← 解析 target 到黑板
│           │   ├── CheckYoloAlive   ← 确认 /vla/yolo_target 可用
│           │   ├── CollectTF2_median← 订阅缓存采 10 帧中值 (RUNNING)
│           │   ├── PlanIK_ABDE      ← IK 航点 A/B/D/E
│           │   └── Sequence [RunMotions]
│           │       ├── OpenClaw
│           │       ├── MoveInit
│           │       ├── MoveReady
│           │       ├── MovePreAim     ← 步骤 A 预瞄
│           │       ├── MoveGrasp      ← 步骤 B 切入
│           │       ├── CloseClaw
│           │       ├── Lift           ← 步骤 D 抬升
│           │       ├── Retreat        ← 步骤 E 关节大展翅
│           │       ├── FoldBack
│           │       ├── MoveHome
│           │       └── ReleaseClaw
│           └── ClearCommand           ← 清空黑板指令
├── Sequence [HandleChat]
│   ├── HasAction[chat]
│   ├── EnsureNotBusy
│   ├── MarkBusy
│   └── BusyScope [ChatBusyScope]
│       └── Sequence [ChatWork]
│           ├── ParseChatFields      ← text/response/reply
│           ├── SpeakTTS             ← POST http://127.0.0.1:5000/tts
│           └── ClearCommand
└── Idle                             ← 无指令时 RUNNING，占住 Selector
```

### 2.2 三态语义（零基础）

| 状态 | 含义 | 本项目中例子 |
|------|------|-------------|
| `SUCCESS` | 本节点任务完成 | 采满 10 帧 YOLO 点 |
| `FAILURE` | 本节点失败 | IK 解算失败、YOLO 超时 |
| `RUNNING` | 还在进行，下 tick 继续 | 采点只收到 3/10 帧；Idle 等待指令 |

**Sequence**：子节点从左到右，一个 `FAILURE` 整段失败；全 `SUCCESS` 才成功。  
**Selector**：从左到右试，一个 `SUCCESS` 就停；全失败才失败。  
**Idle 返回 RUNNING**：让 Selector 在没指令时不空转成功。

### 2.3 与 22.1 状态机的节点映射

| 22.1 `VLAGraspStateMachine` | 22.2 黑板 / BT 节点 |
|----------------------------|---------------------|
| `self.is_working` | `vla/working` + `MarkBusy` / `BusyScope` |
| `command_callback()` | `_on_master_command()` → `vla/command` |
| `execute_grasp_pipeline()` 整体 | `GraspPipeline` 子树 |
| 采 10 帧 TF2 | `CollectYoloTarget` |
| IK A/B/D/E | `PlanGraspTrajectory` |
| 开爪→归位→…→松爪 | `RunMotions` 11 个 `GraspMotion` |
| （无） | `HandleChat` + `SpeakTTS` |

### 2.4 代码文件位置

```text
kuavo-ros-opensource/src/demo/vla_grasp/
├── vla_auto_grasp_daemon.py    # 22.1 状态机版（保留）
├── vla_bt_daemon.py              # 22.2 入口
├── claw_safe.py                  # 夹爪限幅/堵转监测（抓取脚本共用）
├── debug_left_claw.py            # 夹爪扫参/安全探针（见 20.gripper_issue.md）
├── look_down.py                  # 终端 3
├── tts_server.py                 # 终端 4
└── bt/
    ├── blackboard.py             # 黑板键 BBKeys
    ├── config.py                 # ROS 参数默认值
    ├── grasp_skills.py           # IK 规划 + 分步动作（从 daemon 抽出）
```

### 2.5 黑板键一览

| 键名 | 含义 |
|------|------|
| `vla/command` | 待处理 JSON 指令（来自 `/vla/master_command`） |
| `vla/working` | 是否正在执行 grab/chat |
| `vla/target` | 抓取目标名，如「可乐」 |
| `vla/speech` | chat 要播报的文本 |
| `vla/raw_x` / `vla/raw_y` | YOLO 中值坐标 |
| `grasp/plan` | `GraspPlan` 对象（IK 航点） |
| `grasp/skills` | `GraspSkills` 实例 |

### 3.3 导读（与 22.1 图的对照）

1. **纵向分层不变**：Orin 感知决策 → NUC 调度执行 → NUC root 运动底盘 → 物理电机。
2. **唯一换芯点**：原图终端 5 的 `vla_auto_grasp_daemon.py（状态机）` → `vla_bt_daemon.py（py_trees）`。

4. **22.1 踩坑仍有效**：TTS 必须 lab 用户、WBC 须等标定、Ghost_words 在 Orin 终端 9。
5. **夹爪安全**：终端 5 经 `claw_safe.py` 限幅；标定/自锁/探针见 [`20.gripper_issue.md`](./20.gripper_issue.md)。

### 6.3 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `No module named py_trees` | 未安装 | `pip install py_trees` |
| 有指令不抓取 | YOLO 没起 | 查终端 7、`rostopic echo /vla/yolo_target` |
| `CheckYoloAlive` 失败 | ROS 主从不对 / YOLO 断流 | 查 `ROS_MASTER_URI`、重启终端 7 |
| 收到 grab 后 **几十秒** 才动 | YOLO 采点反复失败 + 旧版 command 不清 | 已修复：订阅缓存 + 失败清 command；仍慢则查 YOLO 是否间歇断流 |
| `only got X/10 YOLO samples` | 目标不在视野 / YOLO 频率低 | 调整物体位置；或 `_yolo_sample_count:=6` |
| `timeout waiting first message` | 机器人动完视角变 / YOLO 未发布 | 重启终端 7，确认 `rostopic hz /vla/yolo_target` |
| 没声音 | TTS 用 root 跑 | 终端 4 必须 **lab** 用户 |
| 两个 daemon 都在跑 | 重复订阅 | 只保留一个 |
| ASR「幻觉」连触发 grab | Orin 环境杂音 → LLM 误解析 | NUC 侧 8s grab 防抖；Orin 提高 VAD 阈值（22.1） |
| **左手偏左、右手刚好** | TCP Y 偏置 | 默认 `_tcp_offset_y_left:=-0.015`；仍偏则调到 `-0.02~-0.03` |
| 左爪自锁 / 标定异常 | 启动 homing + 机械 | **见 [20.gripper_issue.md](./20.gripper_issue.md)**，勿在此重复 |

### 6.4 实机调试记录（v49 · 2025-06）

#### 6.4.1 终端 5 日志：第一次 grab 快、第二次慢

**现象**：第一次 `grab`（target=未知）约 1s 采满 10 帧并开始抓取；第二次（target=任意物品）约 **66s** 后才开始，中间大量：

```text
[ERROR] VLA BT: timeout waiting first message on /vla/yolo_target
[ERROR] VLA BT: only got 2/10 YOLO samples
```

**根因**：

1. 旧版 `CollectYoloTarget` 每 tick 用 `wait_for_message(0.15s)` 阻塞，YOLO 间歇断流时极慢；
2. 采点失败后 **command 未清除**，BT 每 tick 重跑整条 `GraspPipeline`，形成「假死重试」；
3. 第二次抓取前机器人已运动，YOLO 可能短暂无目标或频率下降。

**已修复（22.2 当前代码）**：

- YOLO 改为 **Subscriber 缓存最新帧**，非阻塞采集；
- `BusyScope` 在 pipeline **FAILURE 时清除 `vla/command`**，日志：`command cleared (re-issue to retry)`；
- 采点超时默认 **12s**；
- `vla_bt_daemon` 对 grab 指令 **8s 防抖**，减轻 ASR 连触发。

**抓取坐标示例（实机）**：

| 次 | arm | median x | median y | 备注 |
|----|-----|----------|----------|------|
| 1 | left | 0.483 | 0.035 | 偏左 → 调 TCP |
| 2 | right | 0.487 | -0.091 | 落点正常 |

#### 6.4.2 ASR 幻觉与误触发 grab

**现象**：Orin 终端 9 频繁 `录音空白或环境杂音，重新倾听`，仍可能产出 `grab / 任意物品` 等指令。

**NUC 侧**：`GRAB_CMD_MIN_INTERVAL_SEC=8`，8s 内重复 grab 忽略。

**Orin 侧**（在 `rtasr_python3_demo_grasping.py` 调，见 22.1）：提高 VAD/音量阈值、减少环境杂音送 LLM。

#### 6.4.3 左/右抓取 TCP 偏置

TF2 坐标：`y > 0` → 左臂，`y < 0` → 右臂。

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `TCP_OFFSET_Y_LEFT` | **-0.015** | 负值把左爪落点往右（中心线）修 |
| `TCP_OFFSET_Y_RIGHT` | **0.055** | 右臂实测刚好，保持不变 |

例：`raw_y=0.035` → 旧逻辑 `+0.03=0.065`（偏左）→ 新逻辑 `−0.015=0.020`。

#### 6.4.4 夹爪（简述）

VLA 抓取经 `claw_safe.py`：左 close **80 / 0.5A**，堵转监测；以下脚本已统一接入：

- `vla_bt_daemon.py` → `bt/grasp_skills.py`
- `auto_grasp.py` / `auto_grasp_TF2.py` / `moveit_auto_grasp.py` / `moveit_octomap_grasp.py`

抓取前诊断（同一次 launch 内）：

```bash
# NUC root + source devel/setup.bash，WBC 已启动
python3 src/demo/vla_grasp/debug_left_claw.py --side left --safe-lock-test --pause 2
```

启动标定、自锁、临界点探针、退轴等 **详见 [20.gripper_issue.md](./20.gripper_issue.md)**。

#### 6.4.5 采点失败后的正确重试方式

修复后 pipeline 失败会 **自动清除** `vla/command`，终端 5 日志：

```text
VLA BT: grasp pipeline failed, command cleared (re-issue to retry)
```

此时需 **重新发 grab**（Orin 再说一次，或手动）：

```bash
rostopic pub /vla/master_command std_msgs/String \
  '{"data": "{\"action\": \"grab\", \"target\": \"任意物品\"}"}' -1
```

注意：8s 内的重复 grab 会被防抖忽略（防 ASR 幻觉连触发）。
