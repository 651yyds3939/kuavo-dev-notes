### 一、全身控制与优化 (核心，不是独立「案例包」)

> 👉 **通用专题（robotics-notes）**：[ROS 架构逻辑](https://github.com/651yyds3939/robotics-notes/blob/master/robotics/ros_logic.md) · [工作空间文件速查](https://github.com/651yyds3939/robotics-notes/blob/master/robotics/doc_concept.md) · [系统集成](https://github.com/651yyds3939/robotics-notes/blob/master/robotics/robot_system_integration.md) · [全链路导图](https://github.com/651yyds3939/robotics-notes/blob/master/robot_system.md)

| 目录 | 角色 |
| --- | --- |
| `humanoid-control/` | 人形主控核心: MPC/OCS2、WBC、估计、控制器 launch、`humanoid_plan_arm_trajectory` 手臂轨迹、`mobile_manipulator_controllers`、遥控 (含 H12)、部分 `joystick_drivers`、`humanoid_arm_control` (含动捕 IK 打包) 等。 |

### 八、操作、VR、动捕、语音 (节点集合，可组成案例)

`manipulation_nodes/` 下是多 ROS 包，常见用途：

* `motion_capture_ik` : 动捕 IK / VR / MPC 相关（多份 README）。
* `planarmwebsocketservice` : 机械臂 WebSocket / VR API。
* `voice_control_node` : 语音关键词控制。
* `web_video_server` : 图像话题 HTTP 推流。
* `teach_pendant` : 示教器。
* `noitom_hi5_hand_udp_python` : 诺亦腾 HI5 手套 UDP。
* `taiji_trigger_node` : 太极触发。
* `async_web_server_cpp` : 异步 Web/WebSocket 服务 (C++)（顶层另有同名目录时需以实际 `package.xml` 所在为准，避免重复编译泄漏）。
* `handcontrollerdemorosnode` : 手柄演示节点。
* `pico-body-tracking-server` : Pico 身体追踪 (若有)。

*这些是能力模块，具体「案例」要看 launch 与上层应用怎么组合。*

### 总结 (怎么理解「案例」)

* **一整包就是一个「案例」的**：优先看 `demo/`、`pick_basket/`、`roban2_move_boxes/`，以及 `kuavo_humanoid_sdk/examples/`。
* `dynamic_biped/examples/` 里是 文档型/依赖外部仓库 的小型说明案例。
* `humanoid-control/` + `ocs2/` + `kuavo_assets/` + `mujoco|gazebo|isaac` 是 **运行案例的前提基石**，通常不单独叫「某某案例」。
