# Kuavo 机器人二次开发项目可行性指南

本文档基于 `kuavo_all` 工作区内实际代码与官方文档，对原「15 项二次开发矩阵」逐项评估**能否落地、如何落地、需要修正什么**。  
评估对象：`kuavo-ros-opensource`（主栈）、`kuavo-rl-opensource` / `leju_robot_rl`（RL）、本地 demo 与案例文档。

---

## 〇、系统底座（请先读）

### 0.1 实际软件架构（与原稿差异）

| 原稿假设 | 实际情况 |
|---------|---------|
| ROS2 + DDS 双机通信 | 主栈为 **ROS1 Noetic**；双机/多机可用 `ROS_MASTER_URI` 或 `rosbridge`，非必须 ROS2 |
| MoveIt 作为手臂规划核心 | Kuavo **不以 MoveIt 为主**；手臂走 `/ik/*` 服务 + `/kuavo_arm_traj` 等官方接口 |
| 可改底层电机力控 | **核心 HAL/WBC/MPC 为闭源预编译库**（`installed/lib/libros_hardware_plant.so` 等），应用层无法改力矩环 |
| BehaviorTree 需从零引入 | **`src/demo/grab_box` 已有完整 BehaviorTree.CPP v3 工程** |

### 0.2 开源 / 闭源边界

**有源码、可二次开发：**

- 应用 demo、`kuavo_sdk`、`humanoid_controllers` 编排层
- `motion_capture_ik`（IK/FK）、`humanoid_plan_arm_trajectory`（轨迹规划）
- `grab_box` 行为树、`teach_pendant` 示教、`kuavo_arm_collision_check`
- OCS2 框架、配置文件（`.info` / `kuavo.json`）、URDF

**仅二进制、不可改：**

- `hardware_node`、`hardware_plant`、`humanoid_wbc`、`humanoid_interface`
- EtherCAT 电机驱动与 4kHz 级位置 PD 环

### 0.3 算力分工（可选，非强制）

原稿「i9 下位机 + Orin 上位机」可作为**可选**部署方式，并非官方硬性要求。常见分工：

- **工控机（NUC/i9）**：跑 `humanoid_controllers`、IK、行为树、大模型推理
- **扩展计算板（Orin 等）**：相机驱动、ASR、视觉检测，经 ROS 话题回传

无论单机还是双机，**控制接口均为 ROS1 话题/服务**，见 `docs/4开发接口/接口使用文档.md`。

### 0.4 图例

| 标记 | 含义 |
|------|------|
| ✅ | 可直接基于官方能力实现（仿真或真机） |
| ⚠️ | 部分可实现，需降级目标或大量自研 |
| ❌ | 在当前开源接口下不可行 |

## 一、15 项项目矩阵（可行性 + 实现路径）

### 模块 A：视觉-语言-动作 (VLA) 与大模型交互

---

#### 1. 桌面具身 VLA 语义抓取 — ⚠️ 部分可实现

**可实现：** 自然语言 → 目标类别 → 2D 检测 → 3D 位姿 → IK 抓取 → 夹爪闭合，整条链路的**简化版**。

**不可直接照搬原稿：**

- 不要用 MoveIt + `/joint_trajectory_controller`（栈内不存在该主路径）
- 真机手臂仅稳定支持**位置级**控制（`/kuavo_arm_traj`、`/kuavo_arm_target_poses`）

**推荐实现路径：**

```
用户语音/文本
  → LLM 解析目标类别（案例 2 同款 Prompt）
  → YOLOv8 检测 2D 框（官方 yolov8 案例）
  → 深度相机点云 → base_link 下 3D 点
  → /ik/two_arm_hand_pose_cmd_srv 求 IK
  → /kuavo_arm_traj 或 /control_robot_leju_claw 执行抓取
```

**官方参考：**

| 能力 | 路径 |
|------|------|
| YOLO 检测 | `docs/5功能案例/通用案例/yolov8目标检测案例.md` |
| 视觉定位 + IK 抓取完整流程 | `docs/5功能案例/通用案例/二维码抓取水瓶案例.md`（AprilTag 可换 YOLO+深度） |
| 手臂 IK 服务 | `docs/4开发接口/接口使用文档.md` → `/ik/two_arm_hand_pose_cmd_srv` |
| 大模型语音 | `docs/5功能案例/通用案例/接入豆包实时语音大模型案例.md` |

**建议策略：** Stop-and-Go——底盘站定或切经典 WBC 站立，再动手臂；仿真先用 `load_kuavo_mujoco_sim.launch` 验证。

#### 2. 大模型 Prompt 驱动的「空间动作原语提取」 — ✅ 可实现

**可实现：** LLM 输出结构化动作标签 → 正则解析 → 触发 ROS 服务/话题。

**实现方法（修正版）：**

1. 在 System Prompt 中约束输出格式，例如：`[EXECUTE: GRAB(apple)]`、`[EXECUTE: NOD]`。
2. Python 节点订阅 LLM 流式/完整回复，正则提取标签。
3. 映射到**已有** ROS 接口，而非 MoveIt Action：
   - `GRAB` → 调用 IK 服务 + 发布 `/kuavo_arm_traj`
   - `NOD` → 发布 `/robot_head_motion_data`
   - `WALK` → 发布 `/cmd_vel` 或 `/cmd_pose`（需先确认控制器模式）

**官方参考：**

- `docs/5功能案例/通用案例/接入deepseek大模型语音交互案例.md`
- `docs/5功能案例/通用案例/接入豆包实时语音大模型案例.md`
- `src/manipulation_nodes/voice_control_node/`

**工作量：** 低～中（主要是 Prompt + 解析器 + 动作字典映射表）。

#### 3. 桌面异常诊断与状态修复 — ⚠️ 部分可实现

**可实现：**

- 视觉大模型对桌面场景做 JSON 式异常描述（倾倒、悬空、杂乱等）
- 简单动作：语音提醒、标记异常区域、请求人工介入

**难实现 / 需降级：**

- 「扶正水杯」等**复杂接触操作**：无 MoveIt 约束规划，且真机无开放力控，扶正比抓取难一个数量级
- 原稿「Roll 轴 90° 约束 + MoveIt」需改为：IK 多姿态尝试 + 慢速 `/kuavo_arm_traj` + 人工确认

**推荐路径：**

1. 相机截图 → Qwen-VL / 本地 VLM → JSON `{target, status, suggested_action}`
2. `suggested_action == "notify"` → TTS 播报
3. `suggested_action == "grasp_retry"` → 复用案例 1 抓取流程
4. 复杂修复标记为「人工模式」，不要自动执行

### 模块 B：高级控制、操作与安全性

---

#### 6. 基于「虚拟弹簧」的柔性导纳交互 — ⚠️ 部分可实现（需改名降预期）

**准确描述：** 「基于视觉的虚拟弹簧末端跟随」，**不是**真·笛卡尔导纳力控。

**可实现：**

- 深度相机 + MediaPipe 得 $P_{human}$
- `/ik/fk_srv` 读 $P_{robot}$
- $F = k(P_{human} - P_{robot})$，加阻尼得 $\Delta P$，再 IK → `/kuavo_arm_traj`
- **仿真**中效果最佳；真机为「软跟随」，有 100ms 级延迟

**不可实现：**

- 真机接触力感知下的柔顺退缩（无力传感器、无力矩接口）
- 人手**物理拖拽**手臂（见 `18.teaching_gravity_compensation.md` 失败记录）

**官方参考：**

- `src/demo/teleop/teleop_geometric_ik_fusion.py`（MediaPipe + IK，问题见 `question.md`）
- 推荐优先用 `teleop_math_solver.py`（关节映射更稳）
- `docs/5功能案例/通用案例/VR使用开发案例.md`（成熟替代：Quest3 遥操作）

**参数建议：** $k$ 小、$d$ 大；先 Mujoco 仿真 `load_kuavo_mujoco_sim.launch`。

#### 7. 桌面多目标「推移滑动」操作 (Non-prehensile) — ⚠️ 部分可实现

**可实现：**

- PCL/RANSAC 拟合桌面平面
- 生成平行于桌面的末端笛卡尔路径

**修正：**

- 不用 MoveIt Cartesian Path API
- 用 `humanoid_plan_arm_trajectory` 或自写三次样条/贝塞尔 → `/kuavo_arm_traj`
- 力控接触不可控，推速要慢，需急停

**官方参考：**

- `docs/5功能案例/通用案例/手臂轨迹规划案例.md`
- `src/demo/examples_code/hand_plan_arm_trajectory/`

**工作量：** 中（平面分割 + 路径生成 + 碰撞粗检）。

#### 8. 被动推搡对抗与动态足端修正 — ⚠️ 部分可实现

**可实现：**

- 读取 IMU（`/sensors_data_raw`）做倾角阈值监测
- 调用官方**单步/落足点**接口做碎步修正

**官方参考：**

- `docs/5功能案例/通用案例/落足点规划案例.md`
- `src/demo/examples_code/step_control/simStepControl.py`
- RL 部署：`docs/5功能案例/通用案例/强化学习案例lab版.md`、`kuavo-rl-opensource/`

**建议：** 先做「IMU 超阈值 → 触发单步规划」Demo；完整 CP+RL 切换作为进阶研究，需仿真充分验证。

#### 9. 基于点云的桌面 3D 机械臂避障规划 — ⚠️ 部分可实现

**可实现（推荐路线）：**

- 使用 **`kuavo_arm_collision_check`**：基于 URDF/STL 的关节空间碰撞检测，VR 与手臂控制已集成

**原稿路线（Octomap + MoveIt + OMPL）在当前栈不成立：**

- 无 MoveIt `PlanningScene` 主流程
- 需自研 Octomap → 碰撞约束 → IK 迭代，工作量大

**官方参考：**

- `src/kuavo_arm_collision_check/readme.md`
- 启动：`roslaunch kuavo_arm_collision_check arm_collision_check.launch`
- 控制话题改走 `/arm_collision/kuavo_arm_traj`

**工作量：** 中（碰撞包已有；若要做动态点云避障则升为高）。

#### 13. C++ 具身全局行为树调度 (BehaviorTree.CPP) — ✅ 可实现（基于现有 grab_box 扩展）

**不是从零引入，而是扩展官方已有工程。**

**已有能力（`src/demo/grab_box`）：**

- BehaviorTree.CPP **v3**
- 20+ C++ 节点：`GraspBox`（`StatefulActionNode`）、`ForceCheck`、`NavToStart` 等
- XML：`cfg/kuavo_v*/repeat_grab_put_box_in_shelves_with_force.xml`
- Groot 工程：`cfg/kuavo_v*/Project.btproj`
- 100Hz tick + `haltTree` + reset 子树

**推荐扩展路线：**

```
Phase 1  跑通 grab_box 仿真/真机
Phase 2  拆出通用节点库（Nav / Grasp / CheckBattery / LLMCall）
Phase 3  顶层 Fallback：EmergencyStop > BatteryOK > VoiceCmd > TaskTree
Phase 4  LLM 封装为 AsyncActionNode（可调 Python 服务，不必立刻全 C++）
```

**修正原稿：**

- 栈为 **ROS1 + roscpp**，非 ROS2
- 抓取节点应对接 `/ik/*`、`/kuavo_arm_traj`，**不是 MoveIt**
- BT 是**任务编排层**，不替换闭源 WBC/MPC

**官方参考：**

- `docs/5功能案例/综合案例/策略模块搬箱子案例.md`
- `src/kuavo_humanoid_sdk/kuavo_strategy_pytree/`（Python py_trees 并行路线，可逐步迁移）

#### 14. 手势/语音多模态安全密码锁 — ✅ 可实现

**可实现：**

- 行为树根节点 `Sequence`：先 `SafetyGate`，再 `MainTask`
- `SafetyGate`：YOLO 手势 + 语音唤醒词双条件满足 → 置 Blackboard `ACTIVE=true`
- `ACTIVE=false` 时所有运动节点 `FAILURE`，或禁止发布控制话题
- 异常/超时 → 回 `LOCKED`，调用 `StopWalking` / `haltTree`

**官方参考：**

- YOLO：`yolov8目标检测案例.md`（可训练「握拳」类）
- 语音：豆包/deepseek 语音案例中的唤醒逻辑
- BT 门控：`grab_box` 中 `control_bt` / `reset_bt` 服务

**工作量：** 低～中。

#### 15. 局域网全动态 3D 数字孪生可视化面板 — ✅ 可实现（优先用现成工具）

**推荐路线（不必自研 three.js）：**

1. **Foxglove Studio** — 官方日志文档已推荐，支持 rosbag 与 live 可视化
2. **rosbridge + WebSocket** — `docs/4开发接口/kuavo-humanoid-websocket-sdk接口/Websockts通信.md`
3. **RViz** — 工控机本地调试；无桌面时用 SSH -X 或 Foxglove 远程

**自研 Web（three.js + ros3djs）可行但非必须：**

- 广播 `/joint_states`、`/tf`、`/kuavo_arm_traj`
- 加载 `kuavo_assets` 中 URDF

**修正：** 原稿写 ROS2 WebSocket；实际为 **ROS1 `rosbridge_server`**。

**工作量：** 低（Foxglove/rosbridge）；中高（自研 Web 前端）。

## 四、运行注意事项（修正版）

### 4.1 闭源库与编译

- `installed/lib/*.so` 为闭源核心，**禁止**随意 `catkin clean` 删除 install 空间
- 仿真编译见 `docs/4开发接口/仿真环境使用.md`：需 `source installed/setup.zsh` 再 `catkin build humanoid_controllers`

### 4.2 手臂控制铁律

- 真机手臂外部控制：`/arm_traj_change_mode` → `control_mode: 2`
- 仅使用 **位置** 接口：`/kuavo_arm_traj`（`target_torques` 不支持）
- 高风险调试前启用 **案例 14** 安全锁 + 挂绳

### 4.3 示教与数据采集

- 官方示教：`teach_pendant:=1` 的 0-torque 模式（`docs/5功能案例/通用案例/机器人手臂示教案例.md`）
- Pinocchio 重力补偿力矩前馈在真机**不可行**（`18.teaching_gravity_compensation.md`）

### 4.4 传感器与算力

- 深度相机高分辨率长开易导致带宽/CPU 压力，按需单帧触发
- 桌面操作时可暂停激光建图，保留 IMU 供平衡控制器
- 夹爪抓取后避免长时间大电流夹持（官方抓水瓶案例有说明）

### 4.5 网络与多机

- 多机 ROS1：`ROS_MASTER_URI` 指向 master 所在机器 IP
- Web 可视化：`rosbridge_server` 或 Foxglove；与 ROS2 `ROS_DOMAIN_ID` 无关
