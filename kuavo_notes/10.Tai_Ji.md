# 全身打太极动作案例深度解析与机器人控制进阶

## 一、 案例概览与宏观运行逻辑
本案例的核心思想是**“基于绝对时间戳的离线轨迹播放”**。整个系统的运行可以类比为一场“舞台剧”：
* **剧本**：`taiji.json`，规定了绝对时间轴下的每一步走位、躯干位置和双臂 14 个关节的角度。
* **导演**：`action_player.py`，负责读取剧本，把控全局时间，并下发动作指令。
* **演员 A（下半身）**：MPC 步态控制器，负责腿部移动、重心维持和扎马步。
* **演员 B（上半身）**：手臂控制器，负责执行太极的手部动作。

系统并不依赖实时的“走一步算一步”，而是由导演提前规划好未来整套动作，找准起步时间点后，将上下半身的剧本同时下发，确保动作连贯。

## 二、 核心通信机制与看门狗原理

### 1. 模块间通信流程
* **夺取控制权（服务调用）**：程序启动后会调用 `/arm_traj_change_mode` (模式 `2`)，强行关掉机器人自带的摆臂行为，将手臂切换为外部控制。
* **精准对表（话题订阅）**：订阅 `/humanoid_mpc_observation` 获取底层硬件的高精度时钟（`mpc_time`），作为全局唯一参考时间。
* **发放走位图（话题发布）**：导演提前将整套腿部和躯干轨迹一次性发布到 `/humanoid_mpc_foot_pose_target_trajectories`。
* **抓准时机完美同步**：通过监听 `/humanoid_mpc_gait_time_name` 获取步态起步的毫秒级时间 (`gait_start_time`)。当下层时钟走到该时刻，立即下发手臂轨迹 `/kuavo_arm_target_poses`，实现上下半身同频。

### 2. 为什么没有触发“看门狗”停机？
* **实时速度控制**（如 `move_chassis.py`）：类似遥控车，需要不断发送速度指令（如 10Hz 心跳）来“喂狗”，断联即刹车。
* **轨迹目标控制**（本案例）：脚本下发的是长达数十秒的完整时间表。底层控制器接管后自行按表执行，不再需要上层脚本持续发送指令。程序末尾的 `rospy.sleep(max_time)` 是为了等待动作完成而非维持心跳。

## 三、 动作规划与 JSON 剧本编写

### 1. JSON 文件结构
程序利用 `json.load()` 将剧本转化为字典对象，并分为两大核心模块：
* **`step_control`**：包含每步的持续时间、支撑模式（SS/FS/SF）、躯干重心位姿及落脚点坐标。
* **`arm_motion`**：包含动作持续时间及手臂 14 个关节的角度值。

### 2. 生成自定义剧本的方法
* **正向运动学（FK）采集**：开启机器人“零力拖动”模式，手动摆放姿势并记录当前的关节角度数据。
* **动捕映射**：利用 Quest 3 等设备录制真人骨骼位姿（参考 `/leju_quest_bone_poses` 话题），按时间间隔截取并转化。
* **仿真导出**：在 Gazebo 等仿真环境中设定手脚目标点，通过逆解（IK）算出 14 个角度并保存为 JSON。

### 3. 动作连贯性优化
由于关键帧间隔通常较长（1~2 秒），直接执行会导致动作生硬。建议在 Python 脚本中引入**样条插值**，将 1 秒的间隔细分为每 0.05 秒一个小角度，使动作过渡更丝滑。

## 四、 两种架构对比：自定义脚本 vs. 太极脚本
您的脚本代表了**在线计算与命令式控制**，而太极案例代表了**数据驱动与离线缓存**。

| 对比维度 | 自定义合并脚本（`move_chassis` + `smart_grasp`） | 太极案例脚本（`action_player`） |
| :--- | :--- | :--- |
| **架构思想** | **代码驱动**：动作逻辑硬编码在 `for` 循环中，难以排演长舞蹈。 | **数据驱动**：逻辑与数据解耦，更换舞蹈仅需更换 JSON。 |
| **时间同步** | **PC 时钟盲猜**：依赖 `rospy.sleep()`，系统延迟累积会导致上下半身脱节。 | **硬件时钟强锁**：同步底层 `mpc_time`，确保动作与硬件时间戳对齐。 |
| **下半身控制** | **速度遥控**：通过 `/cmd_vel` 盲走，无法确定精准落脚点。 | **空间打点**：指定躯干与落脚点的绝对坐标，实现毫米级走位。 |
| **计算负担** | **在线现算**：表演时实时调用 IK 服务，极耗 CPU，易卡顿。 | **离线缓存**：角度提前算好存入 JSON，表演时零解算压力。 |

## 五、 经典控制的物理限制与保命法则
在缺乏强化学习（RL）的情况下，机器人主要依赖 ZMP（零矩点）理论维持平衡。编排高难度动作时必须遵守：
1. **绝不腾空**：禁止跳跃动作。经典 MPC 在双脚离地瞬间因接触力归零会导致矩阵无解。
2. **先移重心，再抬腿**：若要抬左脚（进入 `FS` 模式），必须先通过 `torso_pose` 将重心压在右脚正上方。
3. **加速度控制**：手臂质量大，挥舞过快产生的离心力会直接破坏底盘平衡。

## 六、 行业前沿参考
宇树（Unitree）或波士顿动力的动作生成流程通常为：**真人动捕采集 -> 运动学重定向 -> 强化学习（RL）追踪**。通过在仿真器中训练神经网络策略，使机器人不仅学会模仿动作，还能在动作过大时通过自动跨步保持平衡。
