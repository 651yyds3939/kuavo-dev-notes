## 第 1 章：采集什么数据？

### 1.1 用途

**模仿学习（Imitation Learning）** 专家示范：Episode 内全自动抓取，Episode 间人工挪瓶。

### 1.2 每帧字段

| 字段 | 来源 | 实机格式 |
|------|------|----------|
| `observation.images.head` | `/camera/color/image_raw` | 1280×800×3 uint8 |
| `observation.state` | `current_joints_rad` | 14 轴 float32 弧度 |
| `action` | `last_commanded_joints_rad` | 14 轴 float32 弧度 |
| `task` | 脚本生成 | `"grasp bottle epN"` |

LeRobot 加载后 image 为 `torch.Size([3, 800, 1280])`（CHW）。

### 1.3 一次 Episode 流程

```text
YOLO 10 帧采点 → 曲肘护胸 → 预瞄(12cm 外) → 水平插入 → 闭爪
    → 垂直抬升（优先 22cm；IK 无解时 18→14→10cm 递减重试）→ 肩膀外摆 75° → 曲肘收手 → init 下垂 → 开爪释放
```

典型 **~1200 帧 @50Hz ≈ 25s**；`raw_y > 0` 左手，`raw_y < 0` 右手。

```text
【下位机 采 raw】  harvester --raw-only  →  lerobot_kuavo_grasp_raw_batch2/
```

> 训练**不需要** ~62GB raw，只拷 **pack 后的** `lerobot_kuavo_grasp_batch2/`。完整命令见 **§15**。

### 2.1 下位机 / 上位机 / 自己电脑各自做什么

| | **下位机** `192.168.26.1` | **上位机** `192.168.26.12` | **自己电脑** |
|--|---------------------------|----------------------------|--------------|
| **定位** | 机器人主控 / ROS Master | 头部相机 + 视觉 | 外接 NVIDIA GPU 训练机 |
| **采数时** | 控制 + 录 raw | 提供图像 + 瓶子位置 | 不参与 |
| **采数后** | 存 raw、删坏条、续采、可当中转 | **pack 成 LeRobot 数据集** | **ACT 训练** |
| **训练** | ❌ | ❌ | ✅ |

**采数阶段（机器人上电，ROS 联网）**

| 机器 | 终端 | 做什么 |
|------|:----:|--------|
| **下位机** | 1 | WBC 站起来 |
| **下位机** | 2 | `move_group.launch` 手臂规划 / IK |
| **下位机** | 3 | `look_down.py` 低头看桌面 |
| **下位机** | 4 | `kuavo_state_publisher.py` → `/joint_states`（14 轴） |
| **上位机** | 5 | `load_robot_head.launch` → `/camera/color/image_raw` |
| **上位机** | 6 | YOLO → `/vla/yolo_target`（瓶子 TF 位置） |
| **下位机** | **7** | **`lerobot_data_harvester.py`**：调 `moveit_auto_grasp` 抓瓶 + 25Hz 录 npz |
| **上位机** | 8（可选） | `rqt_image_view` 看图 |

**一次 Episode 里谁配合谁：**

```text
上位机 YOLO 报瓶子位置 ──ROS──→ 下位机 moveit_auto_grasp 执行整条抓取
                                      ↓ 同时
下位机 harvester 收：相机图 + 关节状态 + 下发动作 → 写 ep_XXXXXX.npz
                                      ↓
Episode 间你人工挪瓶；Episode 内全自动
```

**harvester 录的每帧：**

- 图像：订阅上位机 **`/camera/color/image_raw`**
- `observation.state`：订阅 **`/joint_states`**
- `action`：harvester 自己下发的关节指令

**采数结束后**

| 机器 | 产物 / 操作 |
|------|-------------|
| **下位机** | `lerobot_kuavo_grasp_raw_batch2/`（~62GB，35 条 npz）；删失手、续采、重建 jsonl |
| **上位机** | rsync raw → 跑 `lerobot_pack_dataset.py` → `lerobot_kuavo_grasp_batch2/` |
| **自己电脑** | 只拷 **pack 文件夹** → `lerobot-train`（§15） |

**一句话：** 下位机 = **动起来 + 录 raw**；上位机 = **眼睛（相机/YOLO）+ 整理成训练格式**；自己电脑 = **GPU 学策略**。

## 第 5 章：Harvester CLI 参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `--raw-only` | **开** | NUC 写 npz，不装 lerobot |
| `--episodes` | 100 | Episode 总数 |
| `--output-dir` | `~/...` | **必须绝对路径** `/home/lab/...` |
| `--start-index` | `0` | 续采首 npz 序号，如已有到 ep_000029 则填 `30` |
| `--fps` | 50 | 推荐 **25** 减半写盘与体积 |
| `--no-compress` | 关 | **推荐开**：写盘约 **3~5s/ep**（压缩模式 3~10min） |
| `--reposition-pause` | **`10.0`** | 挪瓶间隔（秒）；可酌情加长 |
| `--use-vision` / `--no-vision` | 开 | YOLO 10 帧采点 |
| `--min-frames` | 50 | 少于此视为失败不保存 |
| `--image-topic` | `/camera/color/image_raw` | 头部 RGB |

**推荐批量采数命令**：

```bash
python3 src/demo/vla_grasp/mcp/lerobot_data_harvester.py \
    --raw-only \
    --episodes 20 \
    --output-dir /home/lab/lerobot_kuavo_grasp_raw_batch2 \
    --fps 25 \
    --no-compress
```

**磁盘估算**：

| 模式 | 单 ep 大小 | 写盘时间 | 10 ep | 100 ep |
|------|-----------|----------|-------|--------|
| 50Hz + zlib | ~1.8GB | 3~10 min | ~1h | ~6~7h |
| 25Hz + `--no-compress` | ~1.5GB | **~3~5s** | ~1 min | ~10 min |

### 5.2 写盘权限（lab 用户）

若 batch 目录曾用 **root** 采过，`lab` 用户写 npz 会 `PermissionError`。修复：

```bash
sudo chown -R lab:lab /home/lab/lerobot_kuavo_grasp_raw_batch2
```

harvester 已加**启动前写权限检查**，无权限会在点火前报错。下位机用 `lab` 或 `root` 均可，**输出目录属主须与运行用户一致**。

### 16.4 失手删除 + 重建 `episodes.jsonl`（必做）

脚本**只检查帧数**，不判断是否真的夹到瓶。失手仍会保存 npz，**打包前必须删**。

**步骤 A — 删 npz**：

```bash

rm -v /home/lab/lerobot_kuavo_grasp_raw_batch2/episodes/ep_000028.npz
```

**步骤 B — 重建 jsonl**（删 npz 后**每次都要跑**，否则 jsonl 行数与磁盘不一致）：

```bash
python3 <<'EOF'
import json, glob, os
raw = "/home/lab/lerobot_kuavo_grasp_raw_batch2"
meta_path = os.path.join(raw, "meta", "episodes.jsonl")

by_file = {}
if os.path.isfile(meta_path):
    with open(meta_path, encoding="utf-8") as f:
        for line in f:
            if line.strip():
                rec = json.loads(line)
                by_file[rec["file"]] = rec

paths = sorted(glob.glob(os.path.join(raw, "episodes", "ep_*.npz")))
records = []
for i, path in enumerate(paths):
    rel = "episodes/" + os.path.basename(path)
    rec = by_file.get(rel, {"task": "grasp bottle", "fps": 25, "num_frames": 0})
    rec = dict(rec)
    rec["episode_index"] = i
    rec["file"] = rel
    records.append(rec)

with open(meta_path, "w", encoding="utf-8") as f:
    for rec in records:
        f.write(json.dumps(rec, ensure_ascii=False) + "\n")

print("npz:", len(paths), "| jsonl:", len(records))
for r in records:
    side = "左" if r.get("is_left_arm") else "右"
    print("  [%02d] %s  %s" % (r["episode_index"], os.path.basename(r["file"]), side))
EOF
```

**步骤 C — 确认两边一致**：

```bash
ls /home/lab/lerobot_kuavo_grasp_raw_batch2/episodes/*.npz | wc -l
wc -l /home/lab/lerobot_kuavo_grasp_raw_batch2/meta/episodes.jsonl
# 两个数字必须相等
```

jsonl 里的 `episode_index` 会重排为 `0 ~ N-1`；**文件名**可带缺口（如缺 009、010），pack 按 jsonl + 实际 npz 读，无影响。
### 16.5 条数怎么算（为何不是 35 = 17+20）

| 误区 | 实际 |
|------|------|
| 「删 2 条后 17 条，再采 20 次 = 37」 | 每次 `--episodes 10` 只加 **10 个文件** |
| 「同一 `--start-index 20` 跑两次 = +20」 | 第二次**覆盖** ep_000020~029，净增 0 |
| 正确累加 | **磁盘上 `ep_*.npz` 个数** = 有效条数 |

**batch2 实机账目（2026-06-28）**：

```text

第 3 批 start-index 30、10/10 成功  →  +10

```

### 15.0 训练是什么（原理速览）

| 问题 | 答案 |
|------|------|
| **是模仿学习吗？** | **是。** 专家示范 + **行为克隆**；学「看到类似画面/状态时，输出和示范接近的 14 轴关节动作」。**不是**强化学习（无 reward、无训练时反复试错）。 |
| **用什么训？** | **PyTorch + LeRobot + ACT 策略** + pack 数据集 |
| **是神经网络吗？** | **是。** ACT = CNN 看图像 + Transformer 预测一段未来动作（action chunk） |
| **必须用 GPU 吗？** | **强烈建议必须。** `--policy.device=cuda`；CPU 极慢，不实用 |
| **8GB 显存够吗？** | **够跑 ACT 试训。** 全分辨率 1280×800 建议 `--batch_size=4`；OOM 再改 `2`。SmolVLA 需 **≥16GB**（§15.7） |

**ACT 在学什么（直觉）：**

```text
输入：头顶相机图 (1280×800) + 当前 14 轴关节角
      ↓  神经网络（CNN + Transformer）
输出：接下来一段关节动作（与示范 action 对齐，最小化 loss）
```

| 层级 | 能证明什么 | 不能证明什么 |
|:----:|------------|--------------|
| 0 | 文件未损坏、能 `from_pretrained` | 策略质量 |
| 1 | 推理管线正常、模型在学示范 | 闭环控制、接触、真抓瓶 |
| 2 | 实时推理延迟、输入话题正常 | 能否完成抓取 |
| 3 | **能不能真抓瓶** | — |

> `lerobot-eval` **不能直接用**：LeRobot 内置环境只有 pusht / aloha 等，**没有 Kuavo gym 环境**。

#### 15.10.2 离线评估脚本（PC 一键跑）

在数据集根目录执行（把 `CKPT` 换成你的 checkpoint 路径）：

```bash
cd "$DATASET_ROOT"          # 例：~/kuavo_all/lerobot_kuavo_grasp_batch2
conda activate lerobot
unset PYTHONPATH

python3 << 'EOF'
import torch
import numpy as np
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.policies.act.modeling_act import ACTPolicy
from lerobot.policies.factory import make_pre_post_processors

DATASET_ROOT = "/home/lwy/kuavo_all/lerobot_kuavo_grasp_batch2"   # 改成你的路径
CKPT = "outputs/train/kuavo_grasp_act_35ep/checkpoints/005000/pretrained_model"
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

policy = ACTPolicy.from_pretrained(CKPT).to(device).eval()
preprocessor, postprocessor = make_pre_post_processors(
    policy_cfg=policy.config,
    pretrained_path=CKPT,
    preprocessor_overrides={"device_processor": {"device": str(device)}},
)

ds = LeRobotDataset("local/kuavo_vla_grasp", root=DATASET_ROOT)
indices = np.linspace(0, len(ds) - 1, 200, dtype=int)

maes = []
for i in indices:
    batch = ds[i]
    obs = {k: v for k, v in batch.items() if k.startswith("observation")}
    gt = batch["action"].float()
    with torch.inference_mode():
        pred = postprocessor(policy.select_action(preprocessor(obs))).squeeze(0).cpu()
    maes.append((pred - gt).abs().mean().item())

print(f"frames={len(ds)}, episodes={ds.num_episodes}")
print(f"Action MAE (rad): mean={np.mean(maes):.4f}, median={np.median(maes):.4f}")
EOF
```

**指标含义：** 在 200 帧均匀采样上，计算模型预测 14 轴关节 action 与专家示范 action 的 **平均绝对误差（MAE，单位 rad）**。越小说明越贴近示范。

## 第 19 章：常见问答（2026-06 答疑）

> 本节整理文档使用过程中反复问到的问题；操作细节仍以 **§2.1、§15** 为准。
### 19.1 这是模仿学习吗？和强化学习有什么区别？

**是模仿学习（Imitation Learning / 行为克隆）。**

- 下位机采的是 **专家示范**：机器人按 `moveit_auto_grasp` 成功抓瓶时的「图像 + 状态 + 动作」
- 训练时网络学：**给定观测 → 输出接近示范的 action**
- **不是**强化学习：训练时没有机器人在环境里试错，也没有 reward 函数
### 19.2 训练必须用 GPU 吗？8GB 显存够不够？

| 项目 | 说明 |
|------|------|
| GPU | **强烈建议 NVIDIA + CUDA**；命令 `--policy.device=cuda` |
| CPU | 理论上能跑，**极慢**，不实用 |
| 8GB + ACT | ✅ **够试训**；1280×800 建议 `batch_size=4`，OOM 改 `2` |
| SmolVLA | ❌ 8GB 不够；建议 **≥16GB** 且数据 **50+ ep** |

自检：`python -c "import torch; print(torch.cuda.is_available())"` 须为 `True`。
### 19.3 下位机和上位机各自做什么？

详见 **§2.1**。简记：

- **下位机**：WBC / MoveIt / 执行抓取 / **写 raw npz**
- **上位机**：相机 / YOLO / **pack 成 LeRobot**
- **自己电脑**：**训练**；不拷 62GB raw，只拷 pack 文件夹
### 19.4 训练用到了什么？训完得到什么？

| 阶段 | 用到什么 | 得到什么 |
|------|----------|----------|
| 训练输入 | pack 数据集 + PyTorch + LeRobot + **ACT 神经网络** | — |
| 训练输出 | — | `checkpoints/.../pretrained_model/`（**策略权重**，`.safetensors`） |
| 不是输出 | — | raw npz、pack 文件夹本身（那是输入） |

训完得到的是 **可部署的策略 checkpoint**（神经网络参数），不是新数据集。实机推理节点 **尚未实现**（§15.8）。
### 19.5 能用 TensorBoard 看训练效果吗？

**LeRobot 当前版本（0.5.x+）默认不支持 TensorBoard 曲线。**

| 方式 | 是否可用 | 说明 |
|------|:--------:|------|
| **终端 loss** | ✅ | `--log_freq=100`，最简单 |
| **WandB** | ✅ 推荐 | `--wandb.enable=true --wandb.project=kuavo_grasp` |
| **TensorBoard** | ❌ | 训练不写 `events.out.tfevents.*`；TB 能启动但页面空白是**预期行为** |

详见 **§15.9**（含「TB 空白页」排查、权重自检、续训 + WandB 命令）。

loss 下降 ≠ 一定能抓瓶；先做 §15.10 离线 MAE，再考虑真机。

### 19.7 怎么测试模型能不能用？必须上真机吗？

**不必先上真机。** 推荐顺序见 **§15.10**：

1. **权重自检**（safetensors 能加载）
2. **PC 离线 MAE**（预测 action vs 专家 action）
3. 续训直到 MAE 明显下降
4. 最后才做真机闭环（需推理节点，**尚未做**）
